# design/mockups

DESIGN-стадия харнесса для GUI-работы (BACKLOG P1-5).

**Правило.** Любые изменения, затрагивающие UI-пути (см. `harness.config.json` → `ui.globs`:
`*.ui`, `*.qml`, каталоги `ui/`, `views/`, `widgets/` и т.п.), требуют, чтобы в ветке был
**одобренный набор из ≥4 стилистически разных мокапов**. Это исполняемый гейт, а не напоминание.

## Структура
```
design/mockups/
  <feature>/
    01-minimal-light.html
    02-dark-pro.html
    03-high-contrast-a11y.html
    04-playful-rounded.html
    NOTES.md
    APPROVED            ← пустой файл-маркер, создаётся ПОСЛЕ ревью
```

## Поток
1. `node hooks/new-mockups.js <feature>` — сгенерирует 4 мокапа-заглушки (4 разных стиля).
2. Доведи их до реальных экранов, покажи ревьюеру, выбери направление.
3. Создай пустой файл `APPROVED` в каталоге фичи.
4. Пиши GUI-код. `hooks/design-gate.js` (в VERIFY/CI) проверит, что гейт закрыт.

## Проверка вручную
```
node hooks/design-gate.js --base main      # 0 = ок, 1 = UI тронут без одобренных мокапов
```

Мокапы намеренно одностраничные HTML — открываются в браузере, легко сравнивать стили,
версионируются в git и не тянут зависимостей.
